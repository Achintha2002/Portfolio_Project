UniFlow
Smart Campus Hub

A comprehensive, production-inspired web system designed to streamline university campus management for both students and staff.

Overview

UniFlow is designed to solve the problem of fragmented and inefficient campus resource management. It centralizes facility and equipment booking, automated schedule conflict detection, and maintenance incident tracking into a single, cohesive platform.

By bringing these critical operations under one roof, UniFlow eliminates double-bookings, speeds up maintenance resolution, and provides a drastically improved experience for the university ecosystem.

Key Features

Automated Booking Management

Conflict detection system preventing overlapping approved reservations for resources. Built with intricate time-series logic.

Maintenance Ticketing

Issue-reporting workflow (Open → In Progress → Resolved → Closed) complete with Azure Blob Storage image attachments.

Role-Based Access Control

Distinct tiers (USER, TECHNICIAN, ADMIN) mapped directly to dynamic UI rendering and robust backend route protection.

Hybrid Authentication

"Sign in with Google" OAuth 2.0 coupled with traditional email/password registration using JWT & BCrypt.

Architecture & Engineering

Design Pattern: The application utilizes a Layered MVC architecture (Controller → Service → Repository → Model) making it highly maintainable and scalable. We utilized ODM via Spring Data MongoDB and DTO patterns to securely transfer data between layers.

Technical Complexities: Managing booking collisions required deep time-series logic in MongoDB. Integrating hybrid security (OAuth2 + traditional JWT) demanded tight coupling with the RBAC system. Implementing Azure Blob Storage native multipart file uploads built a robust pipeline for incident ticketing images.
