Case Study

UniFlow
Smart Campus Hub

A comprehensive, production-inspired web system designed to streamline university campus management for both students and staff.

React 19 Tailwind CSS Vite Java 17 Spring Boot MongoDB Spring Security OAuth 2.0 JWT Azure Blob Storage

Overview

UniFlow was engineered from the ground up to solve the problem of fragmented and inefficient campus resource management. It centralizes facility and equipment booking, automated schedule conflict detection, and maintenance incident tracking into a single, cohesive platform.

By bringing these critical operations under one roof, UniFlow eliminates double-bookings, drastically speeds up maintenance resolution, and provides an exceptionally seamless experience for the entire university ecosystem.

Key Features

Automated Booking

Advanced conflict detection system preventing overlapping approved reservations. Built with intricate time-series logic in MongoDB.

Maintenance Ticketing

Complete issue-reporting workflow (Open → In Progress → Resolved → Closed) supported by Azure Blob Storage for image attachments.

Role-Based Access

Distinct tiers (USER, TECHNICIAN, ADMIN) mapped directly to dynamic UI rendering and robust backend route protection.

Hybrid Auth

"Sign in with Google" OAuth 2.0 coupled seamlessly with traditional email/password registration using JWT & BCrypt.

Architecture & Engineering

Design Pattern: The application leverages a strict Layered MVC architecture (Controller → Service → Repository → Model) ensuring high maintainability and scalability. We utilized ODM via Spring Data MongoDB alongside DTO patterns to securely and efficiently transfer data across layers.

Technical Complexity: Managing booking collisions required deep time-series logic. Integrating hybrid security (OAuth2 + JWT) demanded tight coupling with the RBAC system. Implementing native multipart file uploads directly to Azure Blob Storage established a highly robust pipeline for incident images.
